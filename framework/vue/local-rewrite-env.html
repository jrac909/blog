function getNewEnvConfig 获取并转换 localEnv 格式：

  • DEV -> dev
  • FIN_BETA -> fin-beta
  • dev -> dev

读取 pm2 配置文件内容和 local 配置文件内容，这两个文件是通过调用的时候传入的路径，再获取的。

如果没有 pm2 配置文件, 就直接返回 {} 空对象了 如果有 pm2 配置文件, 读取两个配置文件下对应的 localEnv 配置，匹配规则是 env_${ localEnv }

最终返回合并的配置对象。

rewriteEnv 两个配置文件默认是 app.json 和 localApp.json，e-Commerce 项目是传入的 app.config.js 和 localApp.js,localApp 优先级是 package.json 中的行类配置 > localApp.js 配置 > app.config.js。所以 e-Commerce 的三个文件都配置了 PORT,最后生效的还是 package.json 配置。也就是我们启动后看到的 Proxying 地址。 最后遍历赋值给 process.env 环境变量。
