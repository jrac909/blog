# CMD 和 ENTRYPOINT 的区别

这两个指令都可以执行命令，区别如下：

  • 🐥 CMD: 指定这个容器启动的时候运行的命令，只有最后一个会生效，可被替代
  • 🐥 ENTRYPOINT: 指定这个容器启动的时候要运行的命令, 可以追加命令

# 举例测试 CMD

🐸 1 . 创建 DockerFile 文件，比如：dockerfile-cmd-test，内容如下：

# 一个简单的镜像：启动的时候执行命令 ls -a
FROM centos
CMD ["ls", "-a"]

🐸 2. 构建镜像

# -f 指定镜像文件；-t 指定构建的镜像名（也可以加 tag）
docker build -f docker-file-test -t cmdtest

🐸 3. 启动容器

# 可以看到在启动后，容器直接执行了 ls -a 命令，打印了当前文件夹下的文件信息
docker run cmdtest

TIP

一般来说，我们在执行 run 命令的时候可以同时执行一些命令，比如 docker run -it sh "while(true) do echo loop; done;"

🐸 4. 在启动容器的时候追加命令，比如：

# cmdtest 镜像中有 ls -a 命令，我们想在后面追加 -l 参数，所以在 run 的时候加一个 -l
docker run cmdtest -l

此时会报错。因为 cmd 的清理下 -l 替换了 CMD ["ls", "-a"] 命令, -l 不是命令所以报错。所以，我们可以发现，run 中的命令会替换掉 DockerFile 文件中 CMD 的命令。因此我们想要在 docker run 正常替换，可以如下执行：

# 写一个完整的路径
docker run cmdtest ls -al

# 举例测试 ENTRYPOINT

🐸 1. 创建 DockerFile 文件 dockerfile-entrypoint-test

# 同 CMD，只是指令替换为 ENTRYPOINT
FROM  centos
ENTRYPOINT ["ls", "-a"]

🐸 2. 构建镜像

docker build -f dockerfile-entrypoint-test -t entrypointtest

🐸 3. 启动容器

# 可以看到和 CMD 指令运行结果一样，看不出任何区别
docker run entrypointtest

🐸 4. 在启动容器的时候追加命令

docker run entrypointtest -l

可以看到最终的执行结果是：ls -al，也就是说 -l 参数被追加到了 ls -a 命令后面。
